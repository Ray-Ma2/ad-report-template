# 広告レポートテンプレート - 要件定義書

## プロジェクト概要

### 目的
クライアント向け広告レポートの統一フォーマットを作成し、GitHub Pagesで公開・共有するWebベースのレポートシステムを構築する。

### コンセプト
- **ドリルダウン形式**: 月次サマリー → 月別詳細 → 週別詳細
- **クライアント共有**: URLを共有するだけで最新レポートを閲覧可能
- **印刷対応**: ボタン一つでPDF化・印刷可能

---

## 機能要件

### 1. レポート階層構造

| レベル | 名称 | 内容 |
|--------|------|------|
| Level 1 | 月次サマリー | 複数月の推移を俯瞰、媒体別サマリー |
| Level 2 | 月別詳細 | 特定月の媒体別・キャンペーン別詳細 |
| Level 3 | 週別詳細 | 月内の週次推移、日別データ |

### 2. 表示指標

#### 基本指標（全媒体共通）
| 指標名 | 説明 |
|--------|------|
| Impressions | 表示回数 |
| Clicks | クリック数 |
| CTR | クリック率（Clicks / Impressions × 100） |
| Cost | 広告費用 |
| CPC | クリック単価（Cost / Clicks） |
| CPM | 1,000表示あたりコスト（Cost / Impressions × 1,000） |
| CV | コンバージョン数 |
| CVR | コンバージョン率（CV / Clicks × 100） |
| CPA | 獲得単価（Cost / CV） |

### 3. 対象広告媒体

| 媒体 | 優先度 | 備考 |
|------|--------|------|
| Google Ads | 必須 | 検索・ディスプレイ・動画 |
| Meta (Facebook/Instagram) | 必須 | |
| Yahoo! 広告 | 必須 | 検索・ディスプレイ |
| LINE Ads | 必須 | |
| その他（TikTok等） | 拡張 | 将来的に追加対応 |

### 4. UI/UX要件

#### デザイン
- **カラーテーマ**: 青みベース（プロフェッショナルな印象）
- **レスポンシブ対応**: PC・タブレット・スマホ対応
- **ダークモード**: オプション（将来対応可）

#### ナビゲーション
- パンくずリスト表示
- 前後の月・週への移動リンク
- トップページへの戻りリンク

#### データ表示
- 前期比・前年比の表示（色分け）
- グラフ表示（Chart.js使用）
- ソート・フィルタ機能

### 5. 印刷機能

- 印刷ボタンの設置
- 印刷用CSS（`print.css`）で最適化
- 不要な要素（ナビゲーション等）の非表示
- A4サイズ最適化

---

## 非機能要件

### パフォーマンス
- 初期読み込み: 3秒以内
- 軽量なライブラリのみ使用

### セキュリティ
- 機密データはHTML内に直接埋め込み
- 将来的にはBasic認証 or アクセス制限を検討

### 保守性
- データとテンプレートの分離
- クライアントごとのフォルダ構成
- Git管理による履歴保持

## 制約事項

1. GA4等のアナリティクス指標は今回のスコープ外
2. リアルタイムデータ連携は対象外（手動更新を想定）
3. TikTok等の新媒体は初期テンプレートには含めず、後から追加対応
